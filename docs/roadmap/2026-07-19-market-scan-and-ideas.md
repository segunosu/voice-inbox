# Voice Inbox — Market Scan & Ideas Backlog

_Compiled 2026-07-19 from the build-completion review. Owner action: pick next moves._

## 1. Market scan: does anything duplicate Voice Inbox?

**No single product assembles the full chain** (voice capture → evidence-based multi-project routing → policy-gated execution → answer-back → audit trail). The segments individually are crowding:

| Segment | Nearest products | What they lack |
|---|---|---|
| Capture → organised tasks | VoiceTask AI (voice → to-dos with project tagging); Speakwise (meetings → Notion by project); Voicenotes, AudioPen, Wispr Flow | They file — nothing executes |
| Voice → code | Claude Code native voice dictation (Mar 2026); background agents (Devin, Codex, Claude Code GitHub) taking tickets → PRs | Voice replaces the keyboard, not the desk; no routing/registry |
| Closest single thing | TalkiTo (HN hobby project: voice+Slack for one Claude Code session) | No routing, no policy gates, no audit, single-session |

**Strategic read**: the halves are commodities; the middle — routing + governance + heterogeneous destinations (repos, folders, DMs, sessions) — is the moat. Sources: voicetask.app, speakwiseapp.com, code.claude.com/docs/en/voice-dictation, techsy.io background-agents comparison, HN TalkiTo thread.

## 2. Ideas aligned with the goal (less sitting, more voice)

1. **Audio replies (TTS)** — answers and session reports come back as voice notes; the loop becomes ears-and-mouth. Pennies per reply.
2. **Phone-call capture line** — Twilio number (pay-per-use) to call and just talk; zero unlocking while walking/driving.
3. **Multi-intake splitting** — one long ramble → several captures routed to several projects; walks become planning sessions.
4. **Two-way morning briefing** — digest asks "what matters today?"; the spoken answer queues the day's sessions.
5. **Batch approvals** — all pending Run/Send/Route buttons in one digest message; decisions in one 90-second pass.
6. **Fireflies auto-ingest** — every meeting auto-becomes routed captures (spec Phase-6 meeting mode; key already wired).
7. **The scoreboard** — digest reports "% of captures completed without the PC" vs the 60% target (data already in audit trail).
8. **Self-tuning routing** — complete §11 Stage F: corrections auto-propose aliases; weekly eval re-run via cron.

## 3. Ideas not aligned with the screen-time goal, still valuable

1. **Productise it** — proven market gap + working system + consultancy that sells this: "Voice Inbox for founders" as a Teamsmiths/Deputee multi-tenant Slack offering. Most commercially interesting item here.
2. **TPM user-research line** — parents/coaches speak feedback into a channel/number; routed, clustered, quantified.
3. **Book-by-voice** — dictate chapters on walks (Book - TPU); sessions assemble and edit drafts.
4. **Voice journal** — private project → JOURNAL folder; weekly reflective summaries.
5. **Decision memory / knowledge graph** — "what did I decide about TPM pricing in May?" answered from accumulated intakes (§28 precondition now met).
6. **Content pipeline** — memos → LinkedIn drafts behind a Send button.
7. **Audit trail as a compliance feature** — capture-to-action lineage for regulated-adjacent products (Council Tax, Procurement Deputee).

## Suggested order

1. Scoreboard (keeps the system honest about the real objective)
2. TTS audio replies + phone-call capture (removes the last two screen touches)
3. Productisation (when ready to sell)

_Everything above is buildable on the existing pipeline; items 2.1–2.8 are each roughly an evening._
